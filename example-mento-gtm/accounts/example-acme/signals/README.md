## How these get written

Auto-generated by the Step-5 signal-workflow when a trigger row writes to `signal_events`. The markdown file is the human-readable mirror of the SQL row.

## Why both SQL and markdown

- The SQL row is canonical (used by `v_priority_queue.sql`)
- The markdown file is what Claude reads when answering *"what's happening with Acme?"*
- They never drift because the markdown is generated *from* the SQL row, not edited manually
